## ID Layers

The system tracks **three** ID layers, each owned by a different sheet/app:

| Layer | Owner | Format | Example |
|-------|-------|--------|---------|
| **Project ID** | Producer Dashboard | `PP-YY-NNN` | `PP-26-008` |
| **Episode ID** (Director) | Producer Dashboard | `{Project}-{Type}{NN}` | `PP-26-008-L01` |
| **Production ID** (this app) | Production Booking | `OUT-YYMMDD-PROG-EE` | `AGN-260423-VAD-01` |

When a Producer creates a booking here, they pick the matching **Project ID** from the dropdown (sourced live from the Producer Dashboard sheet) so the Production ID generated by this app is traceable back to its Project upstream.

### Production ID rules

```
TSS  -  260423  -  EXE  -  01
 │         │        │       │
OUT    YYMMDD    PROG    Sequence
```

1. **Immutable** — never rename after creation
2. **Folder-only** — ID on folder name only; files keep original camera names

## Production deploy (Portainer)

Push to `main` or `fix/assign-email-real-results` → GHA workflow `.github/workflows/docker-build.yml` builds + pushes to GHCR with tags `sha-<short>`, `<branch>`, `latest` (main only).

Then in Portainer stack `production-booking` (id 125):

1. Edit `IMAGE_TAG` env to the new `sha-<short>` → **Save settings**
2. Click **Pull and redeploy** (with "Re-pull image" toggled ON)
3. `start.sh` runs on container start — applies prisma db push, backfill SQL, seed, starts the reconciler worker, then `npm start`

Stack endpoint: `http://thestandard.fortiddns.com:9000` (id 2).

```bash
# Local dev with the production compose
cp .env.production .env
docker compose up -d
```

Services: `app` (port 3001 by default) · `nginx` (port 80) · `db` (Postgres 16, internal). The reconciler worker is supervised inside the `app` container by `start.sh`.

See [docs/ops-log.md](docs/ops-log.md) for the deploy journal and [docs/architecture.md#deploy-flow](docs/architecture.md) for what `start.sh` does on boot.

## Email Delivery

The app sends via the signed-in admin's Google account using the Gmail HTTPS API (avoids the SMTP-port block on Render free). Admins must sign out + sign in once after a fresh deploy to grant Gmail send permission.

Optional provider-based delivery still supported (`resend`, `sendgrid`, or SMTP via port `2525` where allowed). See `.env.example`.
